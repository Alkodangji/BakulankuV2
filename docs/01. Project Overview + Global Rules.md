# Aplikasi Toko Mbak UL

## Deskripsi Proyek

Aplikasi Toko Mbak UL merupakan aplikasi desktop berbasis Java Swing yang digunakan untuk membantu pengelolaan transaksi dan operasional toko dalam satu sistem terintegrasi.

Aplikasi dirancang untuk mengelola beberapa aktivitas bisnis utama seperti transaksi BRILink, penjualan BBM, penjualan produk toko, serta pencatatan keuangan.

Sistem menggunakan database MySQL sebagai penyimpanan data utama.

# Modul Utama

## Dashboard

Menampilkan ringkasan informasi dan performa bisnis.

## BRILink

Mengelola transaksi:

* Tarik Tunai
* Setor Tunai / Transfer
* Top Up

## BBM

Mengelola:

* Data BBM
* Penjualan BBM
* Restok BBM

## Penjualan

Mengelola:

* Data Produk
* Penjualan Produk
* Restok Produk

## Keuangan

Mengelola:

* Pemasukan
* Pengeluaran
* Transfer Saldo

## Laporan

Menyediakan laporan dari seluruh modul.

---

# Sistem Saldo

Aplikasi menggunakan dua akun utama:

## Cash

Mewakili uang tunai fisik yang tersedia di toko.

## BRI

Mewakili saldo rekening BRI yang digunakan dalam transaksi BRILink dan transaksi non-tunai lainnya.

---

# Aturan Global Sistem

## Nomor Transaksi

Setiap transaksi memiliki nomor transaksi unik.

Fungsi:

* Identifikasi transaksi
* Pencarian transaksi
* Edit transaksi
* Hapus transaksi
* Laporan

---

## Edit Transaksi

Edit transaksi tidak dilakukan dengan mengganti data secara langsung.

Proses edit dilakukan dengan:

1. Membatalkan efek transaksi lama
2. Menerapkan efek transaksi baru
3. Menyimpan perubahan

Aturan ini berlaku untuk seluruh transaksi yang mempengaruhi saldo atau stok.

---

## Hapus Transaksi

Setiap proses penghapusan transaksi menggunakan Double Confirmation.

### Tahap 1

Konfirmasi penghapusan.

### Tahap 2

Peringatan bahwa penghapusan akan mempengaruhi:

* Saldo
* Stok
* Statistik
* Laporan

---

## Harga Beli

Produk dan BBM menggunakan metode Harga Beli Rata-Rata (Average Cost).

Nilai harga beli pada data master selalu menyimpan harga beli rata-rata terbaru.

---

## Histori Restok

Data restok tetap menyimpan harga beli saat transaksi terjadi.

Tujuannya agar histori tidak berubah walaupun harga beli rata-rata berubah di masa depan.

---

## Edit Data Master

Data Produk dan Data BBM dapat diedit.

Field yang dapat diubah:

* Nama
* Harga Jual
* Harga Beli
* Stok

Perubahan stok melalui edit:

* Tidak dianggap transaksi
* Tidak mempengaruhi saldo
* Tidak membuat histori transaksi baru

Edit digunakan untuk memperbaiki kesalahan data atau menyesuaikan kondisi aktual.

---

## Restok

Restok dianggap sebagai transaksi penuh.

Restok:

* Menambah stok
* Mengurangi saldo akun
* Masuk histori
* Masuk laporan
* Dapat diedit
* Dapat dihapus

---

## Kategori Top Up

Kategori Top Up disimpan dalam tabel master kategori.

Kategori digunakan sebagai referensi saat melakukan transaksi Top Up.
