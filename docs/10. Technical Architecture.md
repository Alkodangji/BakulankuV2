# PROJECT_CONTEXT_PART_10_TECHNICAL_ARCHITECTURE

# Tujuan

Dokumen ini menjelaskan struktur teknis aplikasi dan pola pengembangan yang digunakan selama proses implementasi.

---

# Arsitektur Umum

Aplikasi menggunakan arsitektur sederhana berbasis:

* View
* Model
* DAO
* Database

Tujuan:

* Memisahkan tampilan dan logika data
* Mempermudah maintenance
* Mempermudah pengembangan fitur baru

# Aturan Query

Query database tidak ditulis langsung pada View.

View hanya memanggil DAO.

---

Contoh yang diperbolehkan:

View
↓
DAO
↓
Database

---

Contoh yang tidak diperbolehkan:

View
↓
SQL Langsung
↓
Database

---

# Aturan Validasi

Validasi dilakukan sebelum data dikirim ke DAO.

↓

# Catatan Pengembangan

Prioritas utama aplikasi:

1. Logika bisnis benar
2. Database konsisten
3. CRUD berjalan baik
4. UI rapi

UI modern bukan prioritas utama apabila mengganggu stabilitas sistem.

Fokus utama proyek adalah menghasilkan aplikasi yang stabil, mudah dipahami, dan sesuai kebutuhan bisnis toko.
