# PROJECT_CONTEXT_PART_02_DATABASE

# Struktur Database

Database dibagi menjadi dua kelompok utama:

1. Master Data
2. Data Transaksi

---

# Master Data

Master Data digunakan sebagai data referensi yang digunakan oleh transaksi.

## tb_user

Menyimpan data pengguna aplikasi.

### Fungsi

* Login
* Identitas pengguna sistem

### Relasi

* tb_brilink
* tb_penjualan
* tb_produk_restok
* tb_bbm_penjualan
* tb_bbm_restok
* tb_keuangan

### Kolom

* id_user
* username
* password
* nama

---

## tb_akun

Menyimpan akun saldo yang digunakan sistem.

### Data Awal

* Cash
* BRI

### Fungsi

* Menyimpan saldo akun
* Digunakan pada transaksi keuangan
* Digunakan pada transaksi restok

### Relasi

* tb_produk_restok
* tb_bbm_restok
* tb_keuangan

### Kolom

* id_akun
* nama_akun
* saldo

---

## tb_produk

Menyimpan data produk toko.

### Fungsi

* Referensi transaksi penjualan
* Referensi restok produk

### Relasi

* tb_penjualan_detail
* tb_produk_restok

### Kolom

* id_produk
* kode_produk
* nama_produk
* harga_beli
* harga_jual
* stok
* stok_minimum

### Catatan

Harga beli menyimpan harga beli rata-rata terbaru.

---

## tb_bbm

Menyimpan data BBM.

### Fungsi

* Referensi transaksi penjualan BBM
* Referensi restok BBM

### Relasi

* tb_bbm_penjualan
* tb_bbm_restok

### Kolom

* id_bbm
* kode_bbm
* nama_bbm
* harga_beli
* harga_jual
* stok
* stok_minimum

### Catatan

Harga beli menyimpan harga beli rata-rata terbaru.

---

## tb_kategori_topup

Menyimpan kategori Top Up.

### Fungsi

* Referensi transaksi Top Up

### Relasi

* tb_brilink

### Kolom

* id_kategori
* nama_kategori

# Catatan Database

Seluruh transaksi memiliki:

* Nomor Transaksi
* User Pembuat
* Tanggal Transaksi

Seluruh relasi menggunakan Primary Key dan Foreign Key.

Dokumen ini merupakan dasar untuk pembuatan ERD dan SQL Database.
